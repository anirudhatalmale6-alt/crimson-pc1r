DESIGN NOTES:
- Use green/red Multi-State Indicators (green=ON, red=OFF) for each bit
- Place a text label next to each indicator with the description
- Group by module with section headers
- Add "Back" button to return to Main page

HOW TO BUILD IN CRIMSON 3.2:
1. Open the IO_Summary page in page editor (double-click)
2. From Resource Pane > Primitives tab, drag "Multi-State Indicator" onto page
3. Double-click the indicator to configure:
   - State 0 (OFF): Red color
   - State 1 (ON): Green color
   - Data tag: Select the appropriate tag (e.g., OutputModule3.DATA.0)
4. Add a text label next to each indicator (Text menu > Add, or drag text primitive)
5. Copy/paste indicators for efficiency, just change the tag binding

==============================================================================
                      PAGE 3: ALARM
==============================================================================
Purpose: Display fault/alarm conditions from N7 integer array

Tag: N7 (INT[10] array - 25 fault bits)

--- N7[0] FAULTS (Bits 0-15) ---
Bit 0:  VICE FAIL 2 CLOSE Fault
Bit 1:  Point/Chamfer Motor Overload (MS1/MS2 TRIP)
Bit 3:  Coolant Overload Starter Failure (MS3)
Bit 4:  VICE1 FAIL 2 OPEN Fault
Bit 5:  Stamp Not Home Fault
Bit 6:  Stamp Down Fault
Bit 7:  Beam 1 Not Down Fault
Bit 8:  Beam 2 Not Down Fault
Bit 9:  Beam 1 Up Fault
Bit 10: Beam 2 Up Fault
Bit 11: Beam 1 Not Down Fault
Bit 12: Beam 2 Not Down Fault
Bit 13: Beam 1 Not Up Fault
Bit 14: Beam 2 Not Up Fault
Bit 15: Beam 1 Not Home Fault

--- N7[1] FAULTS (Bits 0-8) ---
Bit 0:  Beam 2 Not Home Fault
Bit 1:  Beam 1 Forward Fault
Bit 2:  Beam 2 Forward Fault
Bit 3:  Beam 1 Not Forward Fault
Bit 4:  Beam 2 Not Forward Fault
Bit 5:  Beam 1 Home Fault
Bit 6:  Beam 2 Home Fault
Bit 7:  No Rods Fault
Bit 8:  VICE2 FAIL 2 OPEN Fault

--- N7[3] and N7[4] ---
N7[3]: Month Saved
N7[4]: Day Saved

ALARM PAGE DESIGN OPTIONS:

OPTION A - Multi-State Indicators (like IO page):
- Use red/green indicators with fault description text
- Red = fault active, Green/dark = no fault
- Tag binding: N7[0].0 through N7[0].15, N7[1].0 through N7[1].8

OPTION B - Built-in Alarm List (RECOMMENDED):
1. Go to Data Tags > PLC1 > Controller_Tags > N7
2. Right-click each bit (N7[0].0, etc.) and configure as Alarm Tag
3. Set alarm description to the fault text listed above
4. On the Alarm page, drag the "Alarm List" primitive from Primitives tab
5. The Alarm List automatically shows all active alarms with timestamps

HOW TO CONFIGURE ALARM TAGS:
1. Navigate to Data Tags > PLC1 > expand N7
2. For each N7 element, set:
   - Trigger: When bit = 1
   - Description: The fault description text
   - Priority: Critical (for safety-related), Warning (for others)
3. The built-in Alarm List primitive will display all triggered alarms

==============================================================================
                    NAVIGATION LINKS
==============================================================================
Each page should have navigation buttons:
- Main page: Buttons to IO_Summary and Alarm pages
- IO_Summary: "Back" button to Main, "Alarms" button to Alarm page
- Alarm page: "Back" button to Main, "IO Status" button to IO_Summary

To create a page link button:
1. Place a Rectangle primitive
2. Add text label ("IO STATUS", "ALARMS", "BACK")
3. Configure Behavior > Go To Page > Select target page
